项目名称: vision_project

一、项目简介
本项目是一个基于视觉伺服的闭环控制原型。系统通过相机采集图像，识别画面中的标定区域与激光点位置，将像素坐标映射到统一画布坐标系，再通过串口发送误差量，驱动下位机进行位置修正。

项目当前重点是主干工程化结构:
1. 用状态机组织运行流程。
2. 用模块化拆分相机、视觉、轨迹、通信和 Web 控制。
3. 保持代码可联调、可扩展、可替换。

二、目录结构说明
camera.py
- 相机采集模块。
- 使用 rpicam-vid 输出 MJPEG 字节流，后台线程持续解析并缓存最新帧。

main.py
- 系统主入口与主状态机。
- 协调全部模块，执行 INIT/HOMING/TRACKING/IDLE/FINISH 流程。

tracker.py
- 视觉处理模块。
- INIT 模式: 检测标定四边形角点。
- TRACKING 模式: 检测激光点并做透视映射。

trajectory.py
- 轨迹规划模块。
- 生成矩形离散轨迹并按顺序输出目标点。
- 提供前瞻快进机制，减少跟踪延迟累积。

uart_comm.py
- 串口通信模块。
- 将 dx, dy 打包为固定协议帧并发送到下位机。

web_stream.py
- Web 推流与控制面板模块。
- 提供视频流页面和命令接口，支持 home/track 指令。

requirements.txt
- Python 依赖列表。

Sample/laser_test.py
- 暗场参数测试脚本。
- 用于验证光斑是否能被阈值稳定分离。

Sample/test_uart.py
- 串口环回测试脚本。
- 用于验证 UART 发送和接收链路。

三、系统运行主流程
1. 启动 main.py。
2. Camera 开始持续采集，Web 服务启动。
3. 状态机从 INIT 开始:
   - 在画面中提取四角点。
   - 角点连续稳定后计算透视矩阵。
   - 终端回车确认后进入 HOMING。
4. HOMING:
   - 目标点固定为画布中心。
   - 检测激光点并映射到画布坐标。
   - 计算误差并经 UART 发送。
   - 误差收敛后进入 IDLE。
5. IDLE:
   - 保持可视化，等待 Web 指令。
6. TRACKING:
   - 根据轨迹控制器输出目标点。
   - 视觉模块持续提供激光映射坐标。
   - 发送误差闭环控制。
   - 目标点完成后进入 FINISH。
7. FINISH:
   - 执行收尾逻辑后返回 IDLE。

四、核心功能拆解
1. 透视标定与坐标统一
- 目标: 将原始图像像素坐标统一映射到 640x480 画布坐标。
- 实现路径:
  1) INIT 模式下检测四边形角点。
  2) 角点稳定后使用 cv2.getPerspectiveTransform 计算矩阵 M。
  3) TRACKING 模式下用 cv2.perspectiveTransform 将激光点从像素系映射到画布系。
- 价值: 下游控制与轨迹规划不再依赖相机安装角度。

2. 激光点检测
- 目标: 在复杂背景中稳定定位激光中心。
- 实现路径:
  1) 在 LAB 空间构建阈值掩膜。
  2) 做形态学闭运算和开运算。
  3) 轮廓面积过滤并取最大候选。
  4) 由轮廓矩计算质心坐标。
- 输出: 激光像素坐标与映射后的画布坐标。

3. 轨迹规划与前瞻快进
- 目标: 输出平滑、连续、可跟踪的离散目标点。
- 实现路径:
  1) 对矩形四边做等距插补。
  2) TrajectoryController 顺序输出目标点。
  3) check_and_fast_forward 在前瞻窗口内允许索引快进。
- 价值: 减少执行滞后导致的重复追点。

4. 串口协议发送
- 目标: 将视觉误差转换为下位机可解析数据帧。
- 实现路径:
  1) dx, dy 放大 10 倍并转 int16。
  2) 按 >BBhh 打包帧头和负载。
  3) 累加和校验后附加帧尾。
- 输出: 固定长度字节帧。

5. Web 观测与指令入口
- 目标: 提供远程可视化与控制入口。
- 实现路径:
  1) /video_feed 推送 MJPEG。
  2) /api/command/<cmd> 接收命令。
  3) main.py 通过队列异步消费命令。
- 价值: 将控制入口与主循环解耦。

五、快速上手
1. 环境准备
- 建议 Python 3.10 及以上。
- 安装依赖:
  pip install -r requirements.txt

2. 硬件准备
- 树莓派已启用相机接口。
- 系统可调用 rpicam-vid。
- 串口设备与端口号正确。

3. 运行项目
- 终端执行:
  python main.py
- 浏览器访问:
  http://<设备IP>:5000

4. 调试顺序建议
1) 先运行 Sample/test_uart.py 验证串口链路。
2) 再运行 Sample/laser_test.py 验证暗场和阈值范围。
3) 最后运行 main.py 联调完整状态机。

六、关键参数与调优入口
main.py
- stable_frames_required: INIT 稳定帧数量。
- stable_jitter_threshold_px: INIT 角点抖动阈值。
- HOMING 阈值: error_dist < 8.0。
- TRACKING tolerance: 15.0。

tracker.py
- Canny 阈值与形态学核大小。
- LAB 通道阈值组合。
- 轮廓面积过滤范围。

trajectory.py
- step_size: 轨迹点间距。
- lookahead_window: 前瞻窗口长度。

uart_comm.py
- port/baudrate 串口配置。
- 协议打包格式与校验策略。

七、二次开发建议
1. 将硬编码参数抽到配置文件。
2. 将日志级别和输出目标配置化。
3. 增加 FINISH 收尾动作实现。
4. 增加单元测试与录制帧回放测试。
5. 为状态机增加超时和异常恢复策略。

八、常见问题定位
1. 无图像
- 检查 rpicam-vid 是否可执行。
- 检查相机接口是否启用。

2. 检测不到激光
- 先看 tracker.py 的调试叠加图。
- 调整 LAB 阈值与面积过滤范围。

3. 串口无输出
- 检查端口占用和权限。
- 用 Sample/test_uart.py 先做环回验证。

4. Web 页面无视频
- 检查 5000 端口可访问性。
- 检查主循环是否持续更新共享帧。

九、当前版本边界
1. 轨迹类型目前为矩形闭环。
2. FINISH 收尾动作仍为预留。
3. 部分参数仍采用代码内常量。
